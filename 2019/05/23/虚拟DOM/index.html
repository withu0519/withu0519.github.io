虚拟DOM

Created2019-05-23|Javascript
|Post View:

1.什么是dom

Document Object Model（文档对象模型）是为html和xml提供的API

按照DOM的标准，html和xml都是以标签为结点构造的树结构，而DOM将html和xml相同的结构本质抽象出来，然后通过脚本语言例如Javascript，按照DOM里的模型标准访问和操作文档内容。

Javascript可以通过DOM直接访问和操作网页文档的内容，连接js和html结构的接口就是DOM

2.为什么操作dom性能开销大

1.dom树的实现模块和js模块是分开的，这些跨模块的通讯增加了成本

2.dom操作引起的浏览器的回流和重绘，使得性能开销巨大

3.虚拟dom

虚拟dom不会立即操作dom，而是将更新的diff内容保存到本地一个JS对象中，最终将这个JS对象一次性attch到dom树上，避免大量无谓计算

4.虚拟dom的意义

1.vdom的真正意义是为了实现跨平台，服务端渲染

2.提供了一个性能不错的dom更新策略

5.虚拟dom 的实现流程

先根据真实dom生成一颗虚拟dom，当虚拟dom某个节点的数据改变后会生成一个新的Vnode，然后Vnode和旧Vnode作对比，发现有不一样的地方就直接修改到真实dom上，然后使旧Vnode的值为Vnode。

diff的过程就是调用名为patch的函数，比较新旧节点，一边比较一边给真实的DOM打补丁。

4.虚拟dom和真实dom的区别

虚拟DOM不会进行排版与重绘操作，虚拟DOM就是把真实DOM转换为Javascript代码

虚拟DOM进行频繁修改，然后一次性比较并修改真实DOM中需要改的部分（注意！），最后并在真实DOM中进行排版与重绘，减少过多DOM节点排版与重绘损耗；

真实DOM频繁排版与重绘的效率是相当低的
