vue3相关知识点

Created2019-03-28|Updated2022-05-27|vue
|Post View:

1.Vue3.0的设计目标是什么？做了哪些优化

设计目标

1.更小 移除了一些不常用的API

2.更快 主要体现在编译方面 diff算法优化、静态提升、事件监听缓存、SSR优化

3.更友好 推出了composition API，大大增加了代码的逻辑组织和代码复用能力

优化方案

1.源码 通过monorepo维护源码 ，为Typescript提供了更好的类型检查

2.性能 体积优化、编译优化、数据劫持优化

3.语法API 组合式API，优化了逻辑组织和代码复用

2.Vue3.0性能提升主要是通过哪几方面体现的？

编译阶段

diff算法优化、静态提升、事件监听缓存、SSR优化

源码体积

除了移除一些不常用的API，更重要的是Tree shanking，任何一个函数如ref、reactive、computed等等，仅仅在使用时才打包，没有用到的模块都被摇掉，打包的整体体积更小

响应式系统

vue2中采用 defineProperty来劫持整个对象，然后进行深度遍历所有属性，给每个属性添加getter和setter，实现响应式

vue3采用proxy重写了响应式系统，因为proxy可以对整个对象进行监听，所以不需要深度遍历

proxy

可以监听动态属性的添加

可以监听到数组的索引和数组的length属性

可以监听到删除属性

3.Vue3.0里为什么要用 Proxy API 替代 defineProperty API ？

Object.defineProperty

Object.defineProperty() 方法会直接在一个对象上定义一个新属性，或者修改一个对象的现有属性，并返回此对象，通过get和set属性来实现响应式，

只能对遍历对象属性进行劫持

Proxy

可以对数据进行劫持，对整个对象进行监听，并返回一个新对象，我们可以只操作新对象来达到响应式的目的

4.Vue3.0 所采用的 Composition Api 与 Vue2.x 使用的 Options Api 有什么不同？

Options Api是一个大型组件，其内部有很多处理逻辑关注点，碎片化使得理解和维护组件变的更加困难，可视化也不强

Composition Api将所有的代码都放到一个函数里，提高了可读性，也有利于后期的迭代和维护

vue2使用mixin文件进行逻辑复用，可能会造成命名冲突和数据来源不清晰，而在vue3中通过Composition Api可以很好地复用

5.说说Vue 3.0中Treeshaking特性？举例说明一下？

Tree shaking是什么

Tree shaking 是一种通过清除多余代码方式来优化项目打包体积的技术，简单来讲，就是在保持代码运行结果不变的前提下，去除无用的代码

Tree shaking如何做

基于es6的 模板语法import和exports，在编译时就能确定模板的依赖关系，以及输入输出的变量

1.编译阶段利用ES6 Module判断哪些模块已经加载

2.判断那些模块和变量未被使用或者引用，进而删除对应代码

有什么作用

  • 减少程序体积（更小）
  • 减少程序执行时间（更快）
  • 便于将来对程序架构进行优化（更友好）
