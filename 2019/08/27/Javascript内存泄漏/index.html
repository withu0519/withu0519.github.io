Javascript内存泄漏

Created2019-08-27|Javascript
|Post View:

1.内存泄漏

内存泄漏是在计算机科学中，由于疏忽或错误造成程序未能释放已经不再使用的内存，这并非指内存物理上的消失，而是应用程序分配某段内存后，由于设计错误，导致在释放该段内存之前就失去了对该段内存的控制，从而造成了资源浪费；对于持续性的服务进程，必须及时释放不再用到的内存，否则内存占用越来越高，轻则影响系统性能，重则直接导致进程崩溃

2.垃圾回收机制

Javascript具有自动垃圾回收机制，也就是说，执行环境会负责管理代码执行过程中使用的内存，通常情况下有两种实现方式

1.标记清除:当变量进入环境时，就标记进入环境，所占用的内存就不能释放，当变量离开环境时，标记离开环境。垃圾回收运行的时候，会标记内存中存储的所有变量，然后它会将所有在上下文中的变量以及被在上下文中的变量引用的变量的标记去掉；在此之后再被加上标记的变量就是待删除的了，原因是在任何上下文的变量中都访问不到它们了；随后垃圾回收程序做一次内存清理，销毁带标记的所有值并收回它们的内存

2.引用计数：语言引擎有一张”引用表”，保存了内存里面所有的资源（通常是各种值）的引用次数。如果一个值的引用次数是0，就表示这个值不再用到了，因此可以将这块内存释放
